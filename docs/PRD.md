# 노션 견적서 웹 뷰어 MVP PRD

## 핵심 정보

**목적**: 노션 데이터베이스에 작성한 견적서를 클라이언트가 별도 계정 없이 고유 URL로 웹에서 확인하고 PDF로 다운로드할 수 있게 한다.
**사용자**: 프리랜서 또는 1인 에이전시가 견적서를 발행하고, 클라이언트가 링크를 통해 견적서를 수신하는 B2B 서비스.

---

## 사용자 여정

```
1. [발행자] 노션 데이터베이스에 견적서 항목 작성
   ↓ 견적서 레코드 저장 후 고유 slug 생성

2. [발행자] 견적서 상세 페이지에서 공유 URL 복사
   ↓ URL 클립보드 복사 완료

3. [발행자] 클라이언트에게 URL 이메일/메시지로 전송
   ↓

4. [클라이언트] 공유 URL 접근 → 견적서 확인 페이지 진입
   ↓ 견적서 존재 여부 확인

   [존재] → 견적서 확인 페이지 → 내용 열람 → PDF 다운로드
   [없음] → 404 Not Found 페이지

5. [클라이언트] PDF 다운로드 버튼 클릭
   ↓ 브라우저에서 PDF 파일 저장

6. [완료] → 견적서 확인/저장 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 노션 API 연동 | Notion API를 통해 데이터베이스의 견적서 레코드를 읽어옴 | 데이터 소스 연결 없이 서비스 불가 | 견적서 확인 페이지 |
| **F002** | 견적서 웹 렌더링 | 노션에서 가져온 데이터를 서비스명/단가/수량/금액/세금/합계 형태로 표시 | 핵심 가치인 견적서 열람 제공 | 견적서 확인 페이지 |
| **F003** | 고유 URL 공유 | slug 기반 고유 URL 생성 및 클립보드 복사 | 클라이언트 전달의 유일한 수단 | 견적서 관리 페이지, 견적서 확인 페이지 |
| **F004** | PDF 다운로드 | 견적서 내용을 PDF 파일로 생성하여 다운로드 | 견적서의 공식 문서화 수단 | 견적서 확인 페이지 |
| **F005** | 견적서 목록 조회 | 노션 데이터베이스의 전체 견적서 목록을 카드 형태로 표시 | 발행자의 견적서 관리 핵심 화면 | 견적서 관리 페이지 |

### 2. MVP 필수 지원 기능

- 견적서 버전 관리 (수정 이력)

- 프로필 상세 관리

## 노션 API 연동 설계

### 노션 데이터베이스 속성 구조 (권장)

노션 데이터베이스에 다음 속성을 설정하여 사용한다.

| 속성명 | 노션 타입 | 설명 |
|--------|----------|------|
| `title` | Title | 견적서 제목 (예: "2025-001 홈페이지 제작") |
| `slug` | Rich Text | 공유 URL용 고유 식별자 (예: "2025-001") |
| `client_name` | Rich Text | 클라이언트 회사명 |
| `client_email` | Email | 클라이언트 이메일 |
| `issue_date` | Date | 발행일 |
| `expiry_date` | Date | 유효기간 |
| `status` | Select | 상태 (초안/발송/승인/만료) |
| `notes` | Rich Text | 비고/특이사항 |
| `items` | Rich Text | JSON 문자열로 견적 항목 배열 저장 |

### 견적 항목 JSON 구조 (items 필드)

```json
[
  {
    "name": "홈페이지 기획 및 디자인",
    "unit_price": 1500000,
    "quantity": 1,
    "tax_rate": 10
  },
  {
    "name": "프론트엔드 개발",
    "unit_price": 2000000,
    "quantity": 1,
    "tax_rate": 10
  }
]
```

### API 호출 흐름

```
클라이언트 브라우저
  → Next.js Server Component (또는 Route Handler)
  → Notion API (GET /v1/databases/{id}/query)
  → 데이터 파싱 및 렌더링
```

- Next.js Server Component에서 직접 Notion API 호출 (API 키 서버 사이드 보호)
- `@notionhq/client` 공식 SDK 사용
- ISR(Incremental Static Regeneration) 또는 `revalidate` 설정으로 캐싱 최적화

## PDF 생성 구현 방향

### 권장 방식: `@react-pdf/renderer`

```bash
npm install @react-pdf/renderer
```

- React 컴포넌트 문법으로 PDF 레이아웃 정의 가능
- 서버 사이드 및 클라이언트 사이드 모두 지원
- 한국어 폰트 임베딩 필요 (Noto Sans KR 또는 Pretendard)

**구현 흐름:**

```
견적서 확인 페이지
  → PDF 다운로드 버튼 클릭
  → /api/invoice/[slug]/pdf Route Handler 호출
  → @react-pdf/renderer로 PDF 버퍼 생성
  → Content-Type: application/pdf 응답
  → 브라우저에서 파일 다운로드
```

### 대안: `html2canvas` + `jsPDF`

```bash
npm install html2canvas jspdf
```

- 화면에 렌더링된 HTML을 캡처하여 PDF 변환
- 구현이 간단하나 해상도 및 레이아웃 정밀도 낮음
- 한국어 처리 문제 적음

**권장**: `@react-pdf/renderer` (인쇄 품질 우선) 또는 Puppeteer 기반 서버 렌더링 (최고 품질, Vercel Edge 제약 있음)

## 데이터 모델

### User (발행자 계정)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| email | 이메일 (로그인 ID) | String |
| name | 발행자 이름 | String |
| created_at | 계정 생성일 | DateTime |

### NotionConfig (노션 연동 설정)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 발행자 FK | → User.id |
| notion_token | Notion Integration Token (암호화) | String |
| database_id | Notion Database ID | String |
| updated_at | 마지막 수정일 | DateTime |

### Invoice (견적서 - 노션에서 파싱된 캐시)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 발행자 FK | → User.id |
| notion_page_id | 노션 페이지 ID | String |
| slug | 공유 URL 식별자 | String (unique) |
| status | 견적서 상태 | Enum |
| cached_at | 마지막 캐시 동기화 시각 | DateTime |

### InvoiceItem (견적 항목)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| invoice_id | 견적서 FK | → Invoice.id |
| name | 서비스/항목명 | String |
| unit_price | 단가 (원) | Integer |
| quantity | 수량 | Integer |
| tax_rate | 세율 (%, 기본 10) | Integer |
