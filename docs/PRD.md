# 노션 견적서 웹 뷰어 MVP PRD

## 핵심 정보

**목적**: 노션 데이터베이스에 작성한 견적서를 클라이언트가 별도 계정 없이 고유 URL로 웹에서 확인하고 PDF로 다운로드할 수 있게 한다.
**사용자**: 관리자(프리랜서/1인 에이전시)가 환경변수로 Notion을 직접 연동하고 견적서를 관리한다. 클라이언트는 전달받은 링크로 견적서를 열람한다.
**운영 방식**: 회원가입/로그인 없음. 관리자는 서버 환경변수로 Notion API를 설정하고, 대시보드에서 URL을 복사해 클라이언트에게 전달한다.

---

## 사용자 여정

```
1. [관리자] 노션 데이터베이스에 견적서 항목 작성
   ↓ 견적서 레코드 저장 후 고유 slug 설정

2. [관리자] 대시보드(/dashboard)에서 견적서 목록 확인
   ↓ 각 견적서의 공유 URL 복사

3. [관리자] 클라이언트에게 URL 이메일/메시지로 전송
   ↓

4. [클라이언트] 공유 URL 접근 → 견적서 확인 페이지 진입
   ↓ 견적서 존재 여부 확인

   [존재] → 견적서 확인 페이지 → 내용 열람 → PDF 다운로드
   [없음] → 404 Not Found 페이지

5. [클라이언트] PDF 다운로드 버튼 클릭
   ↓ 브라우저에서 PDF 파일 저장

6. [완료] → 견적서 확인/저장 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 노션 API 연동 | Notion API를 통해 데이터베이스의 견적서 레코드를 읽어옴 (환경변수 기반) | 데이터 소스 연결 없이 서비스 불가 | 견적서 확인 페이지, 대시보드 |
| **F002** | 견적서 웹 렌더링 | 노션에서 가져온 데이터를 서비스명/단가/수량/금액/세금/합계 형태로 표시 | 핵심 가치인 견적서 열람 제공 | 견적서 확인 페이지 |
| **F003** | 고유 URL 공유 | slug 기반 고유 URL 생성 및 클립보드 복사 | 클라이언트 전달의 유일한 수단 | 대시보드, 견적서 확인 페이지 |
| **F004** | PDF 다운로드 | 견적서 내용을 PDF 파일로 생성하여 다운로드 | 견적서의 공식 문서화 수단 | 견적서 확인 페이지 |
| **F005** | 견적서 목록 조회 | 노션 데이터베이스의 전체 견적서 목록을 카드 형태로 표시 | 관리자의 견적서 관리 핵심 화면 | 대시보드 |

## 노션 API 연동 설계

### 환경변수 설정

Notion API 연동은 서버 환경변수로 직접 설정한다. 별도 UI 설정 화면 없음.

```env
NOTION_TOKEN=secret_xxxx
NOTION_DATABASE_ID=xxxx
```

### 노션 데이터베이스 속성 구조 (권장)

노션 데이터베이스에 다음 속성을 설정하여 사용한다.

| 속성명 | 노션 타입 | 설명 |
|--------|----------|------|
| `title` | Title | 견적서 제목 (예: "2025-001 홈페이지 제작") |
| `slug` | Rich Text | 공유 URL용 고유 식별자 (예: "2025-001") |
| `client_name` | Rich Text | 클라이언트 회사명 |
| `client_email` | Email | 클라이언트 이메일 |
| `issue_date` | Date | 발행일 |
| `expiry_date` | Date | 유효기간 |
| `status` | Select | 상태 (초안/발송/승인/만료) |
| `notes` | Rich Text | 비고/특이사항 |
| `items` | Rich Text | JSON 문자열로 견적 항목 배열 저장 |

### 견적 항목 JSON 구조 (items 필드)

```json
[
  {
    "name": "홈페이지 기획 및 디자인",
    "unit_price": 1500000,
    "quantity": 1,
    "tax_rate": 10
  },
  {
    "name": "프론트엔드 개발",
    "unit_price": 2000000,
    "quantity": 1,
    "tax_rate": 10
  }
]
```

### API 호출 흐름

```
클라이언트 브라우저
  → Next.js Server Component (또는 Route Handler)
  → Notion API (GET /v1/databases/{id}/query)
  → 데이터 파싱 및 렌더링
```

- Next.js Server Component에서 직접 Notion API 호출 (API 키 서버 사이드 보호)
- `@notionhq/client` 공식 SDK 사용
- ISR `revalidate: 60` 설정으로 캐싱 최적화

## PDF 생성 구현 방향

### 권장 방식: `@react-pdf/renderer`

```bash
npm install @react-pdf/renderer
```

- React 컴포넌트 문법으로 PDF 레이아웃 정의 가능
- 서버 사이드 렌더링 지원
- 한국어 폰트 임베딩 필요 (Noto Sans KR)

**구현 흐름:**

```
견적서 확인 페이지
  → PDF 다운로드 버튼 클릭
  → /api/invoice/[slug]/pdf Route Handler 호출
  → @react-pdf/renderer로 PDF 버퍼 생성
  → Content-Type: application/pdf 응답
  → 브라우저에서 파일 다운로드
```

## 데이터 모델

### Invoice (노션에서 파싱된 견적서)

| 필드 | 설명 | 타입 |
|------|------|------|
| slug | 공유 URL 식별자 | String |
| title | 견적서 제목 | String |
| client_name | 클라이언트 회사명 | String |
| client_email | 클라이언트 이메일 | String |
| issue_date | 발행일 | String |
| expiry_date | 유효기간 | String |
| status | 상태 (draft/sent/approved/expired) | Enum |
| notes | 비고/특이사항 | String |
| items | 견적 항목 배열 | InvoiceItem[] |

### InvoiceItem (견적 항목)

| 필드 | 설명 | 타입 |
|------|------|------|
| name | 서비스/항목명 | String |
| unit_price | 단가 (원) | Integer |
| quantity | 수량 | Integer |
| tax_rate | 세율 (%, 기본 10) | Integer |
